# pub-highlights

New developments and publications arrive daily, and it can be difficult to keep up with everything - not only finding the time to read papers but making sure you're not missing any in the first place! This projet is an attempt to lighten the load a bit, by providing a summary of papers published in the last ~week in a given field. ✨

## 🚀 Setup

### Requirements

This project uses [uv](https://docs.astral.sh/uv/) to manage its dependencies. If you don't have uv installed, you can set it up via:
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Additionally, in its current version we rely on [Ollama](https://ollama.com/) to access various open-source LLMs. Before you can run the code in this repository you will need to download and install Ollama for [your operating system](https://ollama.com/download) (available for macOS, Linux, and Windows).

### Installation

To install the project, clone the repository and run:
```bash
git clone git@github.com:atmerritt/pub-highlights.git
cd pub-highlights
uv install
```

<a name="usage-section"></a>
## ▶️ Usage

**Pull over your LLM of choice**

Before running the script, you should decide which LLM to use for the summarization task. The default is set to [`llama3.2:3b`](https://ollama.com/library/llama3.2:3b), but you are free to change this. Regardless of which model you want, you first need to pull it over via Ollama, via:
```bash
ollama pull llama3.2:3b
```

Note that this will take approximately 2 GB of space (exact amount depending on the model).

**Generate publication highlights**

To generate the publication highlights (summaries), run the following:
```bash
uv run pubhighlights <search-term> --n-summaries 3
```

For a full list of options, run:
```bash
uv run pubhighlights --help
```


**Inspect publication highlights**

A markdown file will be generated by the script, containing individual abstract summaries, a high-level summary of all recent publications, and a table containing basic details about each publication.

The default location for these files is the [`latest/`](./latest/) folder, but this can be changed via the `--out-dir` argument for `pubhighlights`. Note that currently, these markdown files are tracked via GitHub, but this can be easily prevented with the `.gitignore` file if necessary.

## 💡 Troubleshooting and known limitations
*valid for v0.0.1*

#### Summarizing several abstracts
The current default model (llama3.2:3b) struggles to summarize more than 3 abstracts. If the number is larger than this, the resulting markdown file will typically be missing the high-level summary and/or show various formatting issues (in paricular, citation references to specific papers). 

This is hopefully something that can be improved by switching models or possibly with further adjustments to the prompt, but as of v0.0.1 we have implemented the `--n-summaries` argument as a workaround. It is recommended to set this value to 3; in this case, you will get the 3 most recent papers summarized, and the full set will be detailed in the markdown table.

#### Summarizing zero abstracts
If the search query you provided does not return any abstracts, the code will not break and will still produce an output markdown file for you. The difference is that (a) the table of papers will be empty and (b) the summary will say something along the lines of "you seem to have forgotten to provide me with any input papers". 

#### arXiv API
The arXiv API has been observed to (occasionally) have some inconsistencies in the number and order of publications that it returns for a given search field. In particular, changing the `--max-results` parameter, which controls how many publications the API will return, can result in a different set of "most recent 3 publications". For this reason we recommend leaving `--max-results` at a very high value (200) so that we can be sure that we have the complete list for a given week and search term.

Every so often you may also encounter a "service unavailable" error from the arXiv API; if this is the case please wait a few minutes and try again - this usually works out.

#### Non-deterministic output via LLMs
The language model used for summarization is stochastic, meaning it can produce slightly different outputs on successive runs for the same input. If you get a summary that seems off or incomplete, try re-running the command.

#### Scientific Scope
Currently the prompt for the LLM assumes that we are only interested in astronomy papers; if this is not the case then some small adjustments to the prompt will be necessary.

#### Operating systems
As of v0.0.1, this code has only been tested on macOS. Users running the code on Linux or Windows may encounter issues, although hopefully not 🤞
